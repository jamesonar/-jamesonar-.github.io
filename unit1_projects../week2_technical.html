Relative, Absolute, and Fixed

Maybe it's because I have delusions of grandure when it comes to my spatial reasoning, but when learning HTML, I found myself very frustrated when trying to apply Margin, Border, and Padding. The concepts seem simple enough, but when applying them to format a website. They sometimes interact in ways you don't expect.First, what does Margin, Border, and Padding mean. The graphic below from Girl Develop It serves as a useful reference point.

Let's start wtih Border. The Border is what you think it is. It deliniates the outer lines of an element with HTML. However, while the Border of an element is generally left invisible, it can just as easily be used to format a page. For example, a border 10 pixels wide, with "groove" design and the color 98bf21 looks like this:

div style="border:10px groove #98bf21"

Per the Girl Develop It diagram, padding can either be used to position cotent within the border (if no color is specified) or it can serve as another frame within the border. We'll use the same code that we did before and notice the difference. Note that 5px corresponds to the top and that the numbers rotate clockwise with the left being padded by 100px. Also note that the padding also affects the size of the border.

div style="border:10px groove #98bf21"
div style="padding: 5px 25px 50px 100px"

Referencing the Girl Develop It diagram again, we'll see the final aspect we'll discuss is the element's margin. Margin is simlar to padding, except you cannot assign color to it and takes place outside fo the border. Margins are commonly used to seperate an element from other components on a page.

div style="border:10px groove #98bf21"
div style="padding: 5px 25px 50px 100px"
div style="margin-top: 100px"
